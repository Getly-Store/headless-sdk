## Add yours

Open a PR that adds **one table row** to "Community projects" (top of the table):

```markdown
| [Project name](https://link-to-it) | One sentence, plain words, what it does | `@getly/mcp`, Next.js, … | [you](https://github.com/you) |
```

Rules — kept short on purpose:

1. **It must exist.** A public URL or repo we can open. Screenshots welcome but not
   a substitute.
2. **It must touch Getly** — the API, a package from this repo, or the MCP server.
3. **One row, one link set.** No marketing paragraphs; the table stays scannable.
4. **Keep your API keys out of the linked code.** We check. A linked repo with a
   committed `getly_sk_live_` key gets the PR closed and the key reported for
   rotation.
5. Closed-source/commercial projects are fine — a live URL is enough.

Maintainers may move standout entries to **Featured** (with a short write-up we'll
ask you to sanity-check first) and feature them on getly.store/developers and in
launch posts — free distribution for you, proof-of-life for us.
